# screen.mdの書き方

## screen.mdの位置づけ

`docs/design.md`の画面設計をもとに、各画面の詳細設計を定義するドキュメント。
実装・レビュー・テストの共通基準として機能する。

## 保存先

`docs/screen.md`

---

## 実行手順

### 1. design.mdから画面情報を収集

`docs/design.md`を読み込み、以下を抽出する。

- 画面一覧（画面ID・画面名）
- 画面遷移図
- 非機能要件（配色・フォント・レスポンシブ等の記載があれば）

### 2. 各画面の詳細を記載

`docs/design.md`から読み取れない情報はユーザーに確認する。

---

## 記載構成

### 1. 画面概要

各画面をリスト形式で記載する。画面ごとにH3見出し（`### SCR-XX 画面名`）を付ける。

```markdown
### SCR-01 ログイン画面

- **画面ID**: SCR-01
- **画面名**: ログイン画面
- **概要**: ログイン時に表示される画面
- **表示要素**: ユーザ名入力欄、パスワード入力欄、ログインボタン
- **挙動・制約**:
  - ユーザ名入力欄: 最大32文字
  - パスワード入力欄: 最小8文字
  - ログインボタン押下: 認証成功時SCR-02へ遷移、認証失敗時エラーメッセージ表示
```

---

### 2. 画面遷移図

mermaidで記載し、遷移条件を矢印に明記する。

```markdown
## 画面遷移図

\`\`\`mermaid
stateDiagram-v2
  [*] --> SCR-01 : アクセス
  SCR-01 --> SCR-02 : ログインボタン押下（認証成功）
  SCR-01 --> SCR-01 : ログインボタン押下（認証失敗）
\`\`\`
```

---

### 3. 配色

用途ごとに16進数で定義する。

```markdown
## 配色

| 用途 | カラーコード |
|------|-------------|
| ベースカラー | #FFFFFF |
| アクセントカラー | #1A73E8 |
| エラー | #D93025 |
| テキスト（本文） | #202124 |
```

---

### 4. フォント

用途別にフォントファミリー・サイズ（px）・ウェイトを記載する。

```markdown
## フォント

| 用途 | フォント | サイズ | ウェイト |
|------|----------|--------|----------|
| 見出し | Noto Sans JP | 24px | 700 |
| 本文 | Noto Sans JP | 16px | 400 |
| ボタン | Noto Sans JP | 14px | 500 |
```

---

### 5. その他制約事項

レスポンシブ・アクセシビリティ・対応ブラウザを数値で明示する。

```markdown
## その他制約事項

- レスポンシブ対応: ブレークポイント 768px（タブレット）、1024px（デスクトップ）
- 対応ブラウザ: Chrome 120+、Firefox 120+、Safari 17+
- アクセシビリティ: WCAG 2.1 AA準拠
```

---

## チェックリスト

- [ ] 画面IDが`design.md`の定義と一致しているか
- [ ] 全画面が画面概要に列挙されているか
- [ ] 表示要素に具体的なUIコンポーネント名が記載されているか
- [ ] 挙動・制約に数値基準が明示されているか（「最大32文字」など）
- [ ] 画面遷移図がmermaidで記載され、遷移条件が明記されているか
- [ ] 配色が16進数で定義されているか
- [ ] フォントがpx単位で定義されているか
- [ ] その他制約事項にレスポンシブのブレークポイントが数値で明示されているか
- [ ] 曖昧な表現がないか（「できれば」「なるべく」「適切な」等）
