# ルール作成ガイド

## ルールファイルの位置づけ

`.claude/rules/` に配置するルールファイルは、Claude Codeが特定のファイルやタスクを扱う際に参照する指示書。
対象ファイル・対象タスクに応じた具体的な記載基準・フォーマット・注意事項を定義する。

## ルールファイルの構成

### frontmatter（任意）

対象ファイルを限定する場合は `path` を指定する。

```markdown
---
path: 対象ファイルパス（例: CLAUDE.md, docs/design.md）
---
```

### 本文構成

1. **位置づけ**: このルールが何を定義するかを1〜2行で記載
2. **書くべき内容**: セクションごとの記載要件を列挙
3. **記載例**: markdownコードブロックで具体例を示す
4. **注意事項**: チェックリスト形式で確認項目を列挙

## 注意事項

- [ ] `path` frontmatterが必要な場合のみ記載されているか
- [ ] 位置づけが1〜2行にまとまっているか
- [ ] 書くべき内容がセクションごとに明確か
- [ ] 記載例が具体的か（抽象的な説明ではなくmarkdownの実例）
- [ ] 注意事項がチェックリスト形式か
- [ ] 曖昧な表現がないか（「できれば」「なるべく」「適切な」等）
- [ ] 既存ルール（`claude.md`, `context.md`, `design.md`）と重複していないか
