# screen.md記載ガイド

## screen.mdの位置づけ

`docs/design.md`の画面設計をもとに、各画面の詳細設計を定義するドキュメント。
実装・レビュー・テストの共通基準として機能する。

## 前提

- `docs/design.md`が存在し、画面一覧・画面遷移図が記載済みであること
- 画面IDは`design.md`の定義と一致させること

## 書くべき内容

### 1. 画面概要

各画面をリスト形式で記載する。

- **画面ID**: `SCR-01`形式で採番
- **画面名**: 画面の名称
- **概要**: 画面の目的を1行で
- **表示要素**: 画面に表示するUIコンポーネント一覧
- **挙動・制約**: 各要素の動作・入力制約・バリデーション

### 2. 画面遷移図

- mermaidの`stateDiagram-v2`または`graph`で記載
- 遷移条件（ボタン押下・イベント名）を矢印に明記
- 全画面IDを網羅する

### 3. 配色

- ベースカラー・アクセントカラー・背景色・テキスト色を16進数で明示
- 用途（ボタン・エラー・成功など）ごとに定義

### 4. フォント

- フォントファミリー・サイズ（px）・ウェイトを用途別に列挙

### 5. その他制約事項

- レスポンシブ対応（ブレークポイント等）
- アクセシビリティ要件
- 対応ブラウザ・デバイス

## 記載例

```markdown
## 画面概要

### SCR-01 ログイン画面

- **画面ID**: SCR-01
- **画面名**: ログイン画面
- **概要**: ログイン時に表示される画面
- **表示要素**: ユーザ名入力欄、パスワード入力欄、ログインボタン
- **挙動・制約**:
  - ユーザ名入力欄: 最大32文字
  - パスワード入力欄: 最小8文字
  - ログインボタン押下: 認証成功時SCR-02へ遷移、認証失敗時エラーメッセージ表示

---

## 画面遷移図

\`\`\`mermaid
stateDiagram-v2
  [*] --> SCR-01 : アクセス
  SCR-01 --> SCR-02 : ログインボタン押下（認証成功）
  SCR-01 --> SCR-01 : ログインボタン押下（認証失敗）
\`\`\`

---

## 配色

| 用途 | カラーコード |
|------|-------------|
| ベースカラー | #FFFFFF |
| アクセントカラー | #1A73E8 |
| エラー | #D93025 |
| テキスト（本文） | #202124 |

---

## フォント

| 用途 | フォント | サイズ | ウェイト |
|------|----------|--------|----------|
| 見出し | Noto Sans JP | 24px | 700 |
| 本文 | Noto Sans JP | 16px | 400 |
| ボタン | Noto Sans JP | 14px | 500 |

---

## その他制約事項

- レスポンシブ対応: ブレークポイント 768px（タブレット）、1024px（デスクトップ）
- 対応ブラウザ: Chrome 120+、Firefox 120+、Safari 17+
- アクセシビリティ: WCAG 2.1 AA準拠
```

## 注意事項

- [ ] 画面IDが`design.md`の定義と一致しているか
- [ ] 全画面が画面概要に列挙されているか
- [ ] 表示要素に具体的なUIコンポーネント名が記載されているか
- [ ] 挙動・制約に数値基準が明示されているか（「最大32文字」など）
- [ ] 画面遷移図がmermaidで記載され、遷移条件が明記されているか
- [ ] 配色が16進数で定義されているか
- [ ] フォントがpx単位で定義されているか
- [ ] その他制約事項にレスポンシブのブレークポイントが数値で明示されているか
- [ ] 曖昧な表現がないか（「できれば」「なるべく」「適切な」等）
