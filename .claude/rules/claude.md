# CLAUDE.md更新ルール

## プロジェクト概要

ソースコードおよびdocsなどを確認し、2～3行程度で短く記載する

## ディレクトリ構成

`ls`等を使用し、ディレクトリ構成を**表形式**で記載する。
ソースコード以外のディレクトリは3階層以上書かず、ソースコードディレクトリはソースファイル手前の階層まで記載する。

以下ディレクトリおよびファイルは実際になくても記載すること。
- `.context/`
- `docs/design.md`: 設計ドキュメント


### 記載例

```markdown
| ディレクトリ | 説明 |
| -- | -- |
| .adr/ | ADR配置場所 |
| .context/ | コンテキストファイル配置場所 |
| .devcontainer/ | DevContainer配置場所 |
| src/main/ | ソースコード |
| src/test/ | テストコード |
```

## 技術スタック

ソースコードおよびdocsなどを確認し、リスト形式で列挙する
主要技術スタックのみを取り出し、10行以内にすること

### 記載例：

```markdown
- **フレームワーク**: FastAPI 0.115+
- **言語**: Python 3.12+
- **ORM**: SQLAlchemy 2.0 (async)
- **バリデーション**: Pydantic v2
- **DB**: PostgreSQL 16
- **パッケージ管理**: uv
```


## コマンド

開発時に使用するコマンドを10個以内で記載する

### 記載例：

```markdown
| コマンド | 用途 |
|---------|------|
| `uv run uvicorn app.main:app --reload` | 開発サーバー起動 |
| `uv run pytest` | テスト実行 |
| `uv run pytest --cov` | カバレッジ付きテスト |
| `uv run ruff check .` | Lint実行 |
| `uv run ruff format .` | フォーマット |
| `uv run alembic upgrade head` | DBマイグレーション適用 |
```


## 最終手順

CLAUDE.mdの作成・修正が完了したら、必ず以下を実行する。

```bash
cp CLAUDE.md AGENTS.md
```

## 注意事項

- [ ] プロジェクト概要は3行以内にまとまっているか
- [ ] ディレクトリ構成は表形式か
- [ ] ソースコード以外のディレクトリ構成は2階層以内で書かれているか
- [ ] ソースコードディレクトリ構成はソースファイル前の階層まで書かれているか
- [ ] `.context`ディレクトリは記載されているか
- [ ] 技術スタックはリスト形式か
- [ ] 技術スタックは10行以内か
- [ ] コマンドは表形式か
- [ ] コマンドは10行以内か
- [ ] 応答原則、行動原則を変更していないか
